# 香山处理器 LSQ 监控可视化方案 - 总结报告

## 1. 项目概述

本项目基于香山处理器(XiangShan)设计文档，设计并实现了一套完整的 **LSQ(Load-Store Queue)监控可视化系统**。该系统可以实时监控LSQ内部状态，帮助理解处理器访存单元的工作原理，辅助性能调优和架构验证。

## 2. LSQ架构分析

### 2.1 核心组件

根据香山处理器设计文档，LSQ包含以下子模块：

```
LSQ (Load-Store Queue)
├── LoadQueue
│   ├── VirtualLoadQueue (64项)     - Load指令顺序维护
│   ├── LoadQueueRAR (32项)         - 读后读违例检测
│   ├── LoadQueueRAW (32项)         - 写后读违例检测
│   ├── LoadQueueReplay (48项)      - Load重发队列
│   ├── LoadQueueUncache            - MMIO/Noncacheable处理
│   └── LoadExceptionBuffer         - 异常处理
└── StoreQueue (48项)               - Store指令队列
```

### 2.2 关键状态信号

| 子模块 | 关键状态 | 描述 |
|--------|----------|------|
| VirtualLoadQueue | allocated, committed, isvec | 分配、提交、向量指令 |
| LoadQueueReplay | allocated, scheduled, blocking, cause | 分配、调度、阻塞、原因 |
| StoreQueue | allocated, addrvalid, datavalid, committed | 分配、地址有效、数据有效、提交 |
| LoadQueueRAR | allocated, released, paddr | 分配、释放标记、物理地址 |
| LoadQueueRAW | allocated, paddr, mask | 分配、物理地址、掩码 |

### 2.3 重发原因分类

LoadQueueReplay支持11种重发原因：

| 代码 | 原因 | 描述 |
|------|------|------|
| C_MA | Store-Load预测违例 | 需要等待Store地址计算 |
| C_TM | TLB Miss | 页表遍历未完成 |
| C_FF | Store数据未准备好 | Forward失败 |
| C_DR | DCache Miss无法分配MSHR | MSHR资源不足 |
| C_DM | DCache Miss | 等待L2 Cache回填 |
| C_WF | 路预测器错误 | 预测失败重发 |
| C_BC | Bank冲突 | DCache Bank忙 |
| C_RAR | LQRAR无空间 | RAR队列满 |
| C_RAW | LQRAW无空间 | RAW队列满 |
| C_NK | ST-LD违例 | Forward检测违例 |
| C_MF | MisalignBuffer满 | 非对齐缓冲满 |

### 3.2 核心功能

#### 3.2.1 实时状态监控

- **队列占用率**: 实时显示各子队列的占用情况
- **状态可视化**: 用颜色编码显示每个entry的状态
- **性能指标**: 入队/出队速率、平均延迟

#### 3.2.2 性能分析

- **吞吐率图表**: Load/Store指令吞吐趋势
- **违例统计**: RAR/RAW违例发生频率
- **重发原因分布**: 各类重发原因的占比

#### 3.2.3 数据流可视化

- **指令流动画**: SVG动画展示指令在LSQ中的流动
- **Forward可视化**: Store-to-Load数据前递过程
- **违例检测**: RAR/RAW违例检测流程

#### 3.2.4 事件日志

- **操作记录**: 记录所有关键操作
- **违例告警**: 高亮显示违例事件
- **筛选功能**: 按事件类型筛选

## 5. 与香山处理器集成

### 5.1 硬件接口

在香山RTL中添加监控模块：

```verilog
module lsq_monitor (
    input clk,
    input rst,
    // VirtualLoadQueue接口
    output [63:0] vlq_allocated,
    output [63:0] vlq_committed,
    // LoadQueueReplay接口
    output [47:0] lqr_allocated,
    output [47:0] lqr_scheduled,
    output [10:0] lqr_cause [47:0],
    // StoreQueue接口
    output [47:0] sq_allocated,
    output [47:0] sq_addrvalid,
    output [47:0] sq_datavalid,
    // ... 其他接口
);
```

### 5.2 数据传输

**WebSocket实时传输**:

```javascript
const ws = new WebSocket('ws://fpga-host:8080/lsq');

ws.onmessage = (event) => {
    const data = JSON.parse(event.data);
    updateQueueStatus(data);
    updateCharts(data);
    addLogEntry(data.event);
};
```

**数据格式**:

```json
{
    "cycle": 12345,
    "vlq": {
        "allocated": [0,1,1,0,1,...],
        "committed": [0,0,1,0,1,...]
    },
    "lqr": {
        "allocated": [0,1,0,0,...],
        "scheduled": [0,1,0,0,...],
        "causes": ["", "C_DM", "", "", ...]
    },
    "events": [
        {"type": "enqueue", "queue": "vlq", "entry": 5}
    ]
}
```

### 5.3 部署方案

**步骤1**: FPGA采集数据
```python
# 通过Firesim或FPGA采集
from firesim import FireSim
sim = FireSim()
lsq_data = sim.get_lsq_state()
```

**步骤2**: WebSocket服务器
```python
import asyncio
import websockets

async def lsq_server(websocket, path):
    while True:
        data = read_lsq_state_from_fpga()
        await websocket.send(json.dumps(data))
        await asyncio.sleep(0.001)  # 1ms采样
```

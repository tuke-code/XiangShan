# 常见问题

1. 如果toffee相关的库找不到，尝试修复执行环境，不要添加任何fake的代码去规避错误。
   如果无法通过环境修复，明确报错给用户。

2. CHANGELOG按时间倒序排列，把新内容放到前面，而不要追加到后面。

3. 对于确认的DUT bug，把触发BUG的条件标记为xfail，避免把整条用例无差别放成预期失败。

4. 提交git时，使用格式化的commit msg。例如："test(memblock): xxxx yyy"，"docs(yyy): xxx zzz"。

5. 新特性完成后，同步编写一份说明文档，描述design与usage，同时在适当的外部文档中添加链接。

6. 代码重构或修改引入外部接口、行为变更时，同步修改已有的设计文档。

7. 新任务完成后，同步更新CHANGELOG。

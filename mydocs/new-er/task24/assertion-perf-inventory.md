# Int ER Task24 Assertion And Perf Inventory

This inventory maps the current integer early-release assertion, debug, perf, and directed-test coverage to the risk list from the sparse UCA spec and the implementation plan. It is intentionally conservative: a risk can be functionally covered while still carrying an open perf or fail-fast coverage gap.

Current result: task24 remains active. The implementation has strong local correctness coverage across UCA, Rename, MEFreeList, ROB/ST, DataPath, and Difftest, but several UCA debug counters and ROB/ST guard conditions are not yet exposed through stable `XSPerfAccumulate` names or dedicated fail-fast checks.

## Remaining Task24 Work

The following items are open before task24 can be closed:

1. Expose stable perf names for UCA debug-only counters needed by system debug: producer ready, readDone decrement, squash decrement, guard decrement, early-free opportunity, early-free issued, commit suppress, generation mismatch or late events, redirect kill, full-untracked allocation, and source duplicate count.
2. Decide and implement the wrong `srcIdx` policy: either add a fail-fast assertion plus negative test for out-of-range `srcIdx`, or update the spec to explicitly accept ROB stale filtering for out-of-range source events.
3. Add an explicit assertion or directed negative test for the spec rule that redirect must not squash a redefiner after it has been treated as redefiner non-speculative or after its old pdest has been early-released.
4. Add or confirm stable fallback reason names for producer-not-ready and pending-interrupt if those fallback paths remain in current scope; otherwise mark them as unsupported in the spec and keep conservative no-early-free behavior.
5. After the remaining source changes are made, rerun the focused module suites for UCA, Rename bundle policy, MEFreeList, DataPath, ROB/ST, and Difftest preprocess before moving to task26.
